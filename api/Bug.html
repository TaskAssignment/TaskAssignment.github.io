Class: Bug

Bug

new Bug()

This represents a generic bug. A bug must have at least title and body. Tags are generated from title and body, based on StackOverflow tags.
Requires:
  • module:mongoose

Requires

  • module:mongoose

Members

(inner) _id :String

This will be the id of this bug. By convension, we use an acronym of the source (2 letters) + the id on the source.
Type:
  • String
Example
A bug with Id 123 from Bugzilla will be saved here with _id = 'BZ123'

(inner) author :String

The person who reported this bug. Not required.
Type:
  • String

(inner) body :String

The complete description of this bug. It's not required if the source doesn't have it.
Type:
  • String

(inner) closedAt :Date

The date this bug was closed on the source database. Not required.
Type:
  • Date

(inner) closedBy :String

The person who closed/solved this bug. Not required.
Type:
  • String

(inner) createdAt :Date

The creation date on the source database! (not this one). Required.
Type:
  • Date

(inner) labels :Array

The labels related to this bug. It's not required.
Type:
  • Array

(inner) parsed :Boolean

This is used for internal purposes only. It checks tags were generated from the title and description.
Type:
  • Boolean
Default Value:
  • false

(inner) status :String

This represents the current status of the bug. It's usually one word that will shortly describe what is happening. If the source has this data, it's also useful.
Type:
  • String

(inner) tags :Array

These tags are generated from tiltle and description. The text is broken into words and those are compared with StackOverflow tags.
Type:
  • Array

(inner) title :String

The title of the bug. Usually is a short description of the problem and is required on most bug trackers. It's also required here.
Type:
  • String

(inner) updatedAt :Date

The last time this was modified.
Type:
  • Date

(inner) url :String

The url to acces this bug on the source website/repository
Type:
  • String
